Pytorch训练聊天机器人遇到的问题

发表于 2019-09-03 | 分类于 好好学习 | 评论数： | 阅读次数：

    最近在跟着PyTorch的教程学习做聊天机器人，在转中文的时候遇到了不少坑，现在记录一下。

问题1：训练时出现loss为nan的情况

    这个错误有很多可能性：

  1. 先检查数据，是否处理后的数据内包含一些非数值类型，导致loss计算出错；
  2. 如果数据没有问题，检查学习率是否过高，在训练过程中，一开始我的学习率设置成1e-3，虽然优化器设置了adam，但在700epoch之后，loss突然跳nan。后面设置成1e-4之后，没有再出现这样的问题；
  3. RNN的梯度爆炸，太常见不过了……加梯度剪切和层归一化（我一开始以为是梯度爆炸，结果加了之后仍然炸，最后检查loss和学习率后，问题解决……）。

问题2：Expected object of backend CPU but got backend CUDA for argument #3 ‘index’

    这个问题的出现很简单……就是有的tensor在CPU上面计算，有的tensor在GPU上面计算，当它们之间要计算的时候……boom。
    解决方案是一个个检查，emmm在这里推荐一个工具-torchsnooper。
    首先执行pip install torchsnooper进行安装，安装完了以后，只需要用 @torchsnooper.snoop() 装饰一下要调试的函数，这个函数在执行的时候，就会自动打印出来每一行的执行结果的 tensor 的形状、数据类型、设备、是否需要梯度的信息，方便检查。

问题3：’lengths’ array has to be sorted in decreasing order

    这个问题是由于padding方法的pad_packed_sequence导致的，因为pad_packed_sequence要求输入向量要按照长度进行排序，没有排序的话就会出现这样错误。

问题4：mask矩阵的设定

    这个问题也困扰我很久，因为计算loss的时候，默认使用0作为padding的mask，而如果padding选用了其他词，计算length就不准确，但是……照样可以收敛，只是length不准确而已。
    不过解决方法也很简单，统一word2index里面的PAD值和传到mask的计算的pad值，确保计算不出现问题……至少强迫症是这么认为的。

其他问题

    主要的问题就是上面4个了，剩下的都是一些小问题，如数据的切分，minibatch的数据选取，mask的计算，batchsize选取等等，反复调试即可解决。

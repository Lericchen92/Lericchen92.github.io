什么是 Python

Python最初是一种解释型的脚本语言，随着版本不断升级，科学计算库不断添加，现在已经成为机器学习、人工智能领域常用的语言之一。

>>> love is not True or False

1、Python的函数参数传递，所有的变量都可以理解是内存中一个对象的“引用”，或者，也可以看似c语言的void*。
Python的类型是属于对象的，而不是变量。

5、Python自省机制，自省就是面向对象的语言所写的程序在运行时，能知道对象的类型，简单一句就是运行时能够获得对象的类型。如type(), dir(), hasattr(), isinstance()等。
6、Python单下划线和双下划线：
__foo__：一种约定，Python内部的名字，用来区别其他用户自定义的命名，以防冲突，就是例如__init__(),__del__(),__call__()这些特殊方法；
_foo：一种约定，用来指定变量私有。程序员用来指定私有变量的一种方式，不能用from module import *导入，其他方面和公有一样访问；
__foo：这个有真正的意义。解析器用_classname__foo来代替这个名字，以区别和其他类相同的命名，它无法直接像公有成员一样随便访问，通过对象名._类名__xxx这样的方式可以访问。
7、迭代器和生成器
这里有个关于生成器的创建问题面试官有问：将列表生成式中[]改成()之后数据结构是否改变？ 答案：是，从列表变为生成器。

通过列表生成式，可以直接创建一个列表。但是受到内存限制，列表容量是有限的。而且，创建一个包含百万元素的列表，不仅占用很大的内存空间，如：我们只需要访问前面的几个元素，后面大部分元素所占的空间都是浪费的。因此，没有必要创建完整的列表（节省大量内存空间）。在Python中，我们可以采用生成器：边循环，边计算的机制—>generator。
8、 *args and **kwargs
当不确定函数将要传递多少参数时可以用*args。
相似的，**kwargs允许使用没有事先定义的参数名。
9、面向切面编程AOP和装饰器
AOP：简言之、这种在运行时，编译时，类和方法加载时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程。
我们管切入到指定类指定方法的代码片段称为切面，而切入到哪些类、哪些方法则叫切入点。有了AOP，我们就可以把几个类共有的代码，抽取到一个切片中，等到需要时再切入对象中去，从而改变其原有的行为。
装饰器：一个很著名的设计模式，经常被用于有切面需求的场景，较为经典的有插入日志、性能测试、事务处理等。装饰器是解决这类问题的绝佳设计，有了装饰器，我们就可以抽离出大量函数中与函数功能本身无关的雷同代码并继续重用。概括的讲，装饰器的作用就是为已经存在的对象添加额外的功能。
10、 Python不支持函数重载，重载要解决的“可变参数类型、可变参数个数”，Python都有解决方案，故不支持。
